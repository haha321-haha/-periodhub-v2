是的，这确实是硬编码问题导致的报错！

## 问题分析

Vercel部署日志显示的都是 `MISSING_MESSAGE` 错误，这表明：

1. **翻译键缺失**：代码中使用了翻译键，但在 `messages/zh.json` 和 `messages/en.json` 文件中找不到对应的翻译内容

2. **涉及的页面**：主要是首页 (`app/[locale]/page.js`)

3. **缺失的翻译键**：
   - `organization.description`
   - `faq.q1.question` / `faq.q1.answer`
   - `faq.q2.question` / `faq.q2.answer`  
   - `faq.q3.question` / `faq.q3.answer`
   - `home.faq.q4.question` / `home.faq.q4.answer`
   - `home.faq.q5.question` / `home.faq.q5.answer`
   - `home.faq.q6.question` / `home.faq.q6.answer`
   - `home.faq.q7.question` / `home.faq.q7.answer`

## 修复方案

需要在翻译文件中补充这些缺失的翻译键：

**在 `messages/zh.json` 中添加：**
```json
{
  "organization": {
    "description": "专业的女性月经健康管理平台"
  },
  "faq": {
    "q1": {
      "question": "痛经怎么缓解最快方法？",
      "answer": "最快的痛经缓解方法包括：热敷下腹部和腰部、轻柔按摩腹部..."
    }
    // ... 其他FAQ
  },
  "home": {
    "faq": {
      "q4": {
        "question": "...",
        "answer": "..."
      }
      // ... q5, q6, q7
    }
  }
}
```

**在 `messages/en.json` 中添加对应的英文翻译**

这个错误正好验证了我们之前讨论的问题：国际化配置已经修复，但翻译内容不完整。建议优先修复这些首页的翻译键，因为它们直接影响用户体验。
