# Meta描述验证和修复计划

## 阶段1：实证验证（优先执行）

### 1.1 字符长度标准验证
**目标：** 确定中英文Meta描述的最佳长度
**方法：** 实际搜索结果测试

**测试页面选择：**
- 中文页面：3-4个现有页面
- 英文页面：3-4个现有页面
- 包含不同长度描述的页面

**测试环境：**
- Google搜索（桌面+移动）
- Bing搜索（桌面+移动）
- 记录实际截断点

**预期发现：**
- 中文：80-120字符 vs 120-140字符
- 英文：150-160字符确认
- 移动vs桌面差异

### 1.2 字段用途澄清
**当前问题字段：**
- `description` - 目的不明确
- `seo_description` - 应该SEO优化但实际没有
- `seo_description_zh` - 不一致

**建议新结构：**
- `description` - 人类可读的页面摘要（长度灵活）
- `meta_description` - 搜索引擎优化（特定语言长度目标）

### 1.3 基线测量记录
**记录内容：**
- 5-10个页面的当前搜索结果外观
- 屏幕截图：描述在实际搜索结果中的显示方式
- 当前点击率和排名数据

## 阶段2：单页测试（风险最低）

### 2.1 测试页面选择
**候选页面：**
1. `ginger-menstrual-pain-relief-guide.md` (30字符 - 最严重)
2. `understanding-your-cycle.md` (41字符 - 影响多字段)
3. `period-pain-simulator-accuracy-analysis.md` (43字符 - 英文问题)

**选择标准：**
- 问题严重程度
- 修复复杂度
- 影响范围

### 2.2 双语言优化
**修复策略：**
- 英文版本：150-160字符目标
- 中文版本：基于验证结果的长度目标
- 确保两个版本都包含关键词和价值主张

### 2.3 生产验证流程
1. **修改** - 更新Meta描述
2. **部署** - 推送到生产环境
3. **验证** - 在搜索结果中确认更改
4. **等待** - 至少48小时让搜索引擎反映更改
5. **监控** - 检查流量/排名影响
