# 浏览器缓存清理指南

## 问题诊断

✅ **服务器端已修复**：通过 `curl` 测试确认服务器返回正确的标题：
- 中文版：`痛经影响算法 - 症状评估与职场分析 | PeriodHub`
- 英文版：`Period Pain Impact Calculator - Symptom Assessment & Workplace Analysis | PeriodHub`

❌ **客户端问题**：浏览器标签页仍显示旧标题，这是典型的浏览器缓存问题。

## 解决方案

### 方法一：强制刷新（推荐）
1. **Chrome/Edge**: `Ctrl+Shift+R` (Windows) 或 `Cmd+Shift+R` (Mac)
2. **Firefox**: `Ctrl+F5` (Windows) 或 `Cmd+Shift+R` (Mac)
3. **Safari**: `Cmd+Option+R` (Mac)

### 方法二：清除浏览器缓存
1. **Chrome**:
   - 按 `F12` 打开开发者工具
   - 右键点击刷新按钮
   - 选择"清空缓存并硬性重新加载"

2. **Firefox**:
   - 按 `Ctrl+Shift+Delete` (Windows) 或 `Cmd+Shift+Delete` (Mac)
   - 选择"缓存"并清除

3. **Safari**:
   - 开发菜单 → 清空缓存

### 方法三：无痕模式测试
- 打开无痕/隐私浏览窗口
- 访问页面确认标题是否正确

### 方法四：使用浏览器控制台脚本
在浏览器控制台运行以下脚本：

```javascript
// 强制设置正确标题
const isChinese = navigator.language.startsWith('zh');
const chineseTitle = '痛经影响算法 - 症状评估与职场分析 | PeriodHub';
const englishTitle = 'Period Pain Impact Calculator - Symptom Assessment & Workplace Analysis | PeriodHub';
const correctTitle = isChinese ? chineseTitle : englishTitle;

document.title = correctTitle;
console.log('标题已强制设置为:', correctTitle);

// 清理可能的缓存
if ('caches' in window) {
  caches.keys().then(names => {
    names.forEach(name => {
      caches.delete(name);
      console.log('已清理缓存:', name);
    });
  });
}
```

## 验证步骤

1. 强制刷新页面
2. 检查浏览器标签页标题
3. 检查页面源代码中的 `<title>` 标签
4. 确认控制台无错误信息

## 如果问题仍然存在

如果按照上述步骤操作后问题仍然存在，可能是：
1. 浏览器扩展干扰
2. 系统级缓存问题
3. 网络代理缓存

请尝试：
- 禁用所有浏览器扩展
- 重启浏览器
- 清除系统DNS缓存
- 使用不同浏览器测试

## 技术说明

此问题是由于浏览器缓存了旧的HTML内容导致的。服务器端已经正确返回新的Meta信息，但客户端浏览器仍在使用缓存的版本。强制刷新会绕过缓存，获取最新的服务器内容。
